# 📄 Ask Your PDF

A **Retrieval-Augmented Generation (RAG)** web app that lets you upload any PDF and ask questions about it in plain English. Answers are generated by Claude and grounded **strictly in the uploaded document**, so responses stay accurate and traceable to the source — if the answer isn't in the PDF, the app says so instead of guessing.

🔗 **Live demo:** https://kristen-rose22-ask-your-pdf-app-7cfbtz.streamlit.app/

> _Tip: the free-tier app sleeps after periods of inactivity — if it shows a "wake up" button, give it ~30 seconds to start._

---

## What it does

Upload a PDF, type a question, and get an answer drawn only from that document. Behind the scenes, the app finds the most relevant passages and asks Claude to answer using just those — keeping responses faithful to the source material.

---

## How it works

The app is built around the standard RAG pipeline, implemented step by step:

1. **Ingestion** — extracts the raw text from the uploaded PDF (`pypdf`).
2. **Chunking** — splits the text into ~200-word overlapping chunks, so related ideas aren't split across boundaries.
3. **Embedding** — converts each chunk into a vector using a local embedding model (`all-MiniLM-L6-v2`) run through ChromaDB.
4. **Storage** — stores the vectors in a ChromaDB vector database.
5. **Retrieval** — embeds the user's question and performs nearest-neighbor search to find the most semantically similar chunks.
6. **Generation** — injects the retrieved chunks into a prompt that instructs Claude to answer *only* from that context, minimizing hallucinations.

A useful way to think about it: **two models, two jobs.** A local embedding model *finds* the relevant text; Claude *writes* the answer.

## Running locally

**Prerequisites:** Python 3.9+ and an [Anthropic API key](https://console.anthropic.com).

```bash
# 1. Clone the repo
git clone https://github.com/kristen-rose22/ask-your-pdf.git
cd ask-your-pdf

# 2. Install dependencies
pip install -r requirements.txt

# 3. Add your API key
#    Create a file at .streamlit/secrets.toml containing:
#    ANTHROPIC_API_KEY = "your-key-here"

# 4. Run the app
streamlit run app.py
```

The first question after launching may take a moment while the embedding model downloads (one time only).

## Design notes

- **Orchestration written directly in Python** (no LangChain or other framework) to keep the retrieval-and-generation flow fully transparent and easy to reason about.
- **API key handled via Streamlit secrets** — never committed to the repository.
- **Answer quality depends on retrieval.** If a wrong answer appears, the first place to look is which chunks were retrieved. Chunk size, overlap, and the number of chunks retrieved (`n_results`) are the main tunable levers.
